## Résumé
Pour terminer la configuration de grabber, on applique une série de commandes pour créer l'utilisateur `grabber`, configurer les fichiers de log dans `/var/log`, ajuster les permissions et installer un répertoire dans `/opt`.

## Commandes et explications

### 1. Création de l'utilisateur et des groupes

`sudo adduser grabber`
- Exécute la création de l'utilisateur `grabber` avec privilèges administrateur (crée généralement le groupe associé automatiquement).

`sudo usermod -aG grabber ramzouille`
- Ajoute l'utilisateur courant `ramzouille` au groupe `grabber` pour faciliter la gestion des fichiers.

`sudo usermod -aG sudo ramzouille`
- Ajoute `ramzouille` au groupe `sudo` (permet l'administration système).

### 2. Configuration des logs (/var/log)

`sudo touch /var/log/grabber-success.log`
- Crée le fichier de log de succès en tant que root.

`sudo touch /var/log/grabber-error.log`
- Crée le fichier de log d'erreurs en tant que root.

`sudo chown grabber:grabber /var/log/grabber*`
- Change propriétaire et groupe des fichiers logs pour `grabber:grabber`.

`sudo chmod u+rw,g+rw,o-rwx /var/log/grabber*`
- Ajuste les permissions : lecture/écriture pour le propriétaire et le groupe, aucun accès pour les autres (privacy).

### 3. Installation dans /opt

`sudo mkdir /opt/grabber`
- Crée le répertoire `/opt/grabber` avec privilèges root.

`sudo chown grabber:grabber /opt/grabber`
- Donne la propriété du dossier `/opt/grabber` à l'utilisateur `grabber`.

### 4. Script et finalisation

`sudo chown grabber:grabber code/grabber/grabber.sh`
- Change le propriétaire du script pour `grabber` afin qu'il puisse l'exécuter/modifier.

`./code/grabber/grabber.sh`
- Exécute le script `grabber.sh` (l'effet dépend du code interne).

`sudo reboot`
- Redémarre la machine pour valider l'ensemble des changements, notamment les groupes.
