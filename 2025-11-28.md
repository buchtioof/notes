date: 2025-11-28

Résumé
-------
Pour terminer la configuration de grabber, on applique une série de commandes pour créer l'utilisateur `grabber`, config les fichiers de log dans `/var/log`, ajuster les permissions et installer un répertoire dans `/opt`.

Commandes et explications (ligne par ligne)

4. `sudo adduser grabber`
   - Exécute la création de l'utilisateur `grabber` avec privilèges administrateur.

7. `sudo addgroup grabber`
   - Crée un groupe `grabber` (équivalent de `groupadd`).

8. `sudo getent group`
   - Affiche les entrées de la base des groupes (permet de vérifier membres et existence).

9. `sudo usermod -aG grabber grabber`
   - Ajoute l'utilisateur `grabber` au groupe `grabber` sans supprimer ses autres groupes (`-aG`).

10. `sudo usermod -aG grabber ramzouille`
	- Ajoute `ramzouille` soit "user" au groupe `grabber`.

11. `sudo getent group`
	- Re-vérification des groupes pour confirmer les changements.

12. `sudo usermod -aG sudo ramzouille`
	- Ajoute `ramzouille` au groupe `sudo`, lui donnant la possibilité d'utiliser `sudo` (sur les systèmes qui l'utilisent).

13. `ls -l /var/log`
	- Liste détaillée du répertoire `/var/log` pour voir permissions, propriétaires et tailles.

14. `cd /var/log`
	- Se déplace dans le répertoire `/var/log`.

15. `touch grabber-success.log`
	- Tente de créer `grabber-success.log` dans `/var/log` ; sans `sudo` cela échouera généralement à cause des permissions.

16. `sudo touch grabber-success.log`
	- Crée le fichier `grabber-success.log` en tant que root.

17. `sudo touch grabber-error.log`
	- Crée le fichier `grabber-error.log` en tant que root (destiné aux erreurs).

18. `ls -l /var/log`
	- Vérifie la création des fichiers et leurs propriétaires.

19. `sudo chown grabber:grabber grabber*`
	- Change propriétaire et groupe des fichiers commençant par `grabber` pour `grabber:grabber`.

20. `ls -l /var/log`
	- Vérifie le changement de propriétaire.

21. `sudo chmod u+rw grabber*`
	- Donne au propriétaire (user) les droits lecture/écriture sur `grabber*`.

22. `sudo chmod g+rw grabber*`
	- Donne au groupe les droits lecture/écriture.

23. `sudo chmod o-rwx grabber*`
	- Retire toutes les permissions aux autres (privacy — pas d'accès world).

24. `ls -l /var/log`
	- Vérifie l'état final des permissions.

25. `cd`
	- Retour au répertoire personnel de l'utilisateur (`$HOME`).

26. `ls`
	- Liste le contenu du répertoire personnel.

27. `cd code/grabber/`
	- Se positionne dans le répertoire du projet (chemin relatif depuis le répertoire personnel).

28. `ls`
	- Liste les fichiers du répertoire `code/grabber/`.

29. `sudo chown grabber:grabber grabber.sh`
	- Change le propriétaire du script `grabber.sh` pour `grabber`, afin que cet utilisateur puisse le modifier/exécuter selon les permissions.

30. `ls -l`
	- Vérifie le propriétaire et les permissions du script.

31. `ls /opt/`
	- Liste le contenu du répertoire `/opt` (emplacement courant pour installer des applications additionnelles).

32. `cd /opt/`
	- Se rend dans `/opt`.

33. `ls`
	- Liste le contenu après s'être déplacé.

34. `mkdir grabber`
	- Tente de créer `grabber` dans `/opt` en tant qu'utilisateur courant — cela échouera normalement sans sudo.

35. `sudo mkdir grabber`
	- Crée `/opt/grabber` avec privilèges root.

36. `sudo chown grabber:grabber grabber`
	- Donne la propriété du dossier `/opt/grabber` à l'utilisateur `grabber`.

37. `cd`
	- Retour au répertoire personnel.

38. `cd code/grabber/`
	- Retour dans le répertoire du projet.

39. `./grabber.sh`
	- Exécute le script `grabber.sh`. L'effet dépendra des permissions et des actions codées dans le script.

41. `sudo reboot`
	- Redémarre immédiatement la machine en tant que root.

Remarques et bonnes pratiques
----------------------------
- Préférer `sudo` pour les actions nécessitant des droits root (création dans `/opt`, modification de `/var/log`, reboot).
- Vérifier le contenu de `grabber.sh` avant exécution pour s'assurer qu'il est sûr et que les chemins (logs, `/opt/grabber`) sont corrects.
- Pour automatiser proprement, on peut créer un petit script d'installation ou un playbook Ansible qui :
  - crée l'utilisateur/groupe, 
  - crée et sécurise les fichiers de logs,
  - installe `/opt/grabber` et règle les permissions,
  - ajoute `ramzouille` au groupe `sudo` si voulu.
